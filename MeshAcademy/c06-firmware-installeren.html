Sectie 1 van 3

MeshCore Firmware

MeshCore is open-source firmware ontwikkeld door Ripplebiz, speciaal ontworpen voor LoRa mesh-netwerken. Het transformeert gewone ESP32- of nRF52-boards tot krachtige mesh-nodes die zonder internet of telefoonnetwerk kunnen communiceren.

Waarom aangepaste firmware?

Een standaard ESP32-board kan niet deelnemen aan een mesh-netwerk. MeshCore-firmware voegt essentiële functionaliteit toe:

  • LoRa mesh-routering — berichten worden automatisch doorgestuurd via meerdere hops
  • End-to-end encryptie — elk gesprek is versleuteld met unieke sleutels
  • Node-rollen — je kiest of het apparaat een client, repeater of room server wordt
  • Energiebeheer — ultra-laag stroomverbruik voor langdurig gebruik op batterij

Waar download je firmware?

MeshCore-firmware wordt gepubliceerd op GitHub (Ripplebiz/MeshCore). De eenvoudigste manier is via de online web-flasher op flasher.meshcore.co.uk, die automatisch de juiste firmware voor je board downloadt.

Firmware-varianten per apparaat

📦

Heltec V3 / WiFi LoRa 32

ESP32-S3 gebaseerd. Populairste instapmodel. Beschikbaar als Companion, Repeater en Room Server firmware.

📦

LilyGO T-Beam

ESP32 met ingebouwde GPS. Ideaal voor mobiel gebruik en locatiebepaling binnen het netwerk.

📦

LilyGO T-Deck

Standalone apparaat met scherm en toetsenbord. Draait als zelfstandige client zonder telefoon.

📦

RAK WisBlock (nRF52840)

Ultra-zuinig nRF52840 platform. Beste keuze voor zonne-energie repeaters door extreem laag verbruik.

Versienummering

MeshCore-releases volgen een versienummer zoals v1.14.1. Controleer altijd dat je de nieuwste stabiele versie flasht. De web-flasher toont automatisch de laatste release. Op GitHub kun je ook oudere versies en release notes bekijken.

ℹ️
Tip: De web-flasher op flasher.meshcore.co.uk herkent automatisch je boardtype en biedt alleen compatibele firmware-varianten aan. Je hoeft niet zelf het juiste bestand te zoeken.
Sectie 2 van 3

Firmware Flashen

Flashen betekent het schrijven van nieuwe firmware naar het geheugen van je board. Er zijn twee methoden: de web-flasher (aanbevolen) of handmatig met esptool (Python).

Methode 1: Web-flasher (aanbevolen)

De snelste en eenvoudigste manier. Geen installatie nodig — alleen een Chrome of Edge browser.

1

USB aansluiten

Verbind je board via USB-C (of micro-USB) met je computer.

2

Web-flasher openen

Ga naar flasher.meshcore.co.uk in Chrome of Edge (andere browsers ondersteunen Web Serial niet).

3

Board en rol kiezen

Selecteer je boardtype (bijv. Heltec V3) en de gewenste firmware-rol (Companion, Repeater, Room Server).

4

Verbinden

Klik "Connect" en selecteer de juiste COM-/seriële poort in het browserdialoog.

5

Flashen

Klik "Flash" — de firmware wordt automatisch gedownload en naar het board geschreven. Dit duurt 1–2 minuten.

6

Herstart controleren

Na het flashen herstart het board automatisch met MeshCore. Je ziet een bevestiging in de web-flasher.

Methode 2: esptool (handmatig)

Voor gevorderde gebruikers of als de web-flasher niet werkt. Vereist Python en esptool:

  1. Installeer Python 3 en vervolgens pip install esptool
  2. Download het juiste firmware-bestand (.bin) van GitHub
  3. Verbind het board via USB
  4. Zet het board in bootloader-modus: houd de BOOT-knop ingedrukt en druk kort op RST (Reset)
  5. Voer het flash-commando uit: esptool.py --port /dev/ttyUSB0 write_flash 0x0 firmware.bin
  6. Na het flashen: druk op RST om het board te herstarten
⚠️
Belangrijk: Flash altijd de juiste firmware voor jouw specifieke boardtype. Een verkeerd firmware-bestand kan je apparaat onbruikbaar maken. De web-flasher voorkomt dit automatisch, maar bij handmatig flashen moet je zelf controleren.

Veelvoorkomende problemen

  • Poort niet gevonden: Controleer de USB-kabel (sommige kabels zijn alleen voor opladen, niet voor data). Probeer een andere USB-poort.
  • Board reageert niet: Zet het board handmatig in bootloader-modus (BOOT + RST).
  • Permission denied (Linux): Voeg je gebruiker toe aan de dialout-groep: sudo usermod -aG dialout $USER
  • Flash mislukt halverwege: Probeer opnieuw. Houd de BOOT-knop ingedrukt tijdens het hele flashproces bij hardnekkige problemen.
💡
Tip: Na succesvol flashen kun je direct de MeshCore-app op je telefoon verbinden via Bluetooth (bij Companion-firmware) om te controleren of alles werkt.
Sectie 3 van 3

Node Rollen

Bij het flashen kies je een rol voor je node. De rol bepaalt hoe het apparaat zich gedraagt in het mesh-netwerk. Dit is de belangrijkste keuze die je maakt.

🔁

Repeater / Relay

De ruggengraat van het netwerk. Stuurt berichten door zonder menselijke interactie. Geen scherm of Bluetooth nodig — alleen de radio. Ideaal voor vaste locaties met zonne-energie op hoge punten.

📱

Client (Companion)

Eindgebruiker-apparaat dat via Bluetooth (BLE) of USB verbindt met je telefoon of computer. Je stuurt en ontvangt berichten via de MeshCore-app. Clients herhalen GEEN berichten.

🏠

Room Server

Host een chatroom in het netwerk. Slaat berichten op zodat gebruikers die later online komen ze alsnog kunnen lezen. Fungeert als centraal punt voor groepscommunicatie.

📡

Base Station

Een beheerde repeater die extra telemetrie rapporteert: batterijniveau, signaalsterkte en uptime. Ideaal voor netwerkbeheerders die de gezondheid van het netwerk willen monitoren.

Wanneer kies je welke rol?

  • Repeater — als je een vaste node plaatst om het bereik van het netwerk te vergroten (bijv. op een dak of mast)
  • Client — als je zelf berichten wilt versturen en ontvangen via je telefoon
  • Room Server — als je een groepskanaal wilt hosten waar meerdere mensen berichten kunnen achterlaten
  • Base Station — als je een beheerde repeater wilt met monitoringfunctie

Vergelijking van rollen

Rol Functie Stuurt door? Stroomverbruik Typisch apparaat
Repeater Berichten doorsturen ✓ Ja Laag (geen BLE/scherm) Heltec V3, RAK WisBlock
Client Berichten zenden & ontvangen ✗ Nee Gemiddeld (BLE actief) Heltec V3, T-Beam
Room Server Chatroom hosten & opslaan ✓ Ja (groep) Gemiddeld Heltec V3, RAK WisBlock
Base Station Beheerde repeater + telemetrie ✓ Ja Laag Heltec V3, RAK WisBlock
ℹ️
Belangrijk: Anders dan bij Meshtastic herhalen clients in MeshCore geen berichten. Alleen repeaters, room servers en base stations sturen door. Dit voorkomt ongewenste flooding en maakt het netwerk stabieler. Sinds firmware v1.14 kun je op een companion wel een ad-hoc repeater modus inschakelen — bedoeld voor gebruik in het veld zonder repeaters in de buurt. Dit geeft tijdelijk een vergelijkbare werking als Meshtastic.
🔮
Vooruitblik: De rol kies je tijdens het flashen. Na het flashen configureer je de overige instellingen (frequentie, zendvermogen, node-naam) — dat behandelen we in de volgende cursus: Eerste Configuratie.
