# PLAN.md ‚Äî Presentatie "Mei MeshCore Maand"

## Projectoverzicht

Een multi-page interactieve HTML/SVG/JS presentatie voor de kick-off van **Mei MeshCore Maand** bij Hackerspace Drenthe. De presentatie wordt vertoond op **woensdag 6 mei 2026** en dient als enthousiasmerende introductie voor een breed publiek inclusief gemeente en pers.

**Kernboodschap:**
> Bij dit project gaat het om verbinding in de breedste zin van het woord. Letterlijk, door een robuust radionetwerk te bouwen dat functioneert wanneer internet en mobiele masten uitvallen. Maar ook figuurlijk, door een hechte community te smeden van doeners die klaarstaan voor hun regio.

### De drie pijlers

De hele presentatie is opgebouwd rond drie pijlers die als rode draad door elke slide lopen:

| Pijler | Betekenis | In de presentatie |
|---|---|---|
| üîó **Verbinden** | Letterlijk: radiogolven die apparaten koppelen. Figuurlijk: mensen die elkaar vinden rond een gezamenlijk doel. | Elke technische uitleg wordt gekoppeld aan het maatschappelijke doel ‚Äî communicatie wanneer het ertoe doet. |
| ü§ù **Samenwerken** | Dit is geen solo-project. Het netwerk werkt alleen als we het samen bouwen ‚Äî net zoals de nodes in het mesh alleen samen een dekkend netwerk vormen. | Rollen, hack-avonden, open source, community. De kracht zit in het collectief. |
| üéì **Leren** | Iedereen begint ergens. Van solderen tot RF-propagatie ‚Äî we leren door te doen, samen. De MeshCore Academy maakt kennis toegankelijk voor elk niveau. | MeshAcademy als gratis leerplatform, workshops op de hak-avonden, kennisdeling. |

Deze drie pijlers vormen samen de identiteit van het project: **we bouwen niet alleen een netwerk van apparaten, maar ook een netwerk van mensen die van elkaar leren.**

## Taalrichtlijnen ‚Äî "Snapt een 16-jarige dit?"

Het hele verhaal moet te begrijpen zijn voor iemand van 16 die niks weet van radio of netwerken. Elke technische term wordt ge√Øntroduceerd met een vergelijking uit het dagelijks leven *voordat* het vaktwoord valt.

| Principe | Richtlijn | Voorbeeld |
|---|---|---|
| **Vergelijk eerst, benoem dan** | Leg het concept uit met een herkenbaar beeld, geef daarna pas de technische term | "Berichten springen van kastje naar kastje ‚Äî dat heet *mesh*." |
| **Korte zinnen** | Max ~15 woorden per zin op slides. Spreektekst mag langer, maar √©√©n idee per zin. | |
| **Geen afkortingen zonder introductie** | SNR, RF, dBm, ERP ‚Äî allemaal eerst uitleggen in gewone taal | "Hoe sterk het signaal boven de ruis uitkomt ‚Äî dat noemen we SNR." |

| **Stapel niet** | Introduceer max 2 nieuwe termen per slide | |
| **Dagelijkse vergelijkingen** | Mesh = estafette, LoRa = fluisteren over lang veld, repeater = versterker in een telefoonketen, antenne = trechter voor geluid | |

- Licht op donker (consistent met website-thema)

### Overgangen

- **Simpele fade** voor alle slide-wisselingen (300ms, ease-in-out)

## SOLID Architectuur

### S ‚Äî Single Responsibility
- Elke **slide** is een eigen HTML-bestand in `slides/`, geladen door de engine (multi-page architectuur)
- Elke **animatie** is een eigen herbruikbaar JS/SVG-bestand in `animations/`
- **CSS** is gescheiden: base-presentatie in `css/`, component-stijlen per animatie
- **Engine** (navigatie, slide-transities, presenter-modus) is een los bestand

### O ‚Äî Open/Closed
- Animaties zijn configureerbaar via data-attributen en parameters
- Nieuwe slides toevoegen zonder bestaande engine of animaties te wijzigen
- Spreker-secties kunnen onafhankelijk worden (her)geordend

### L ‚Äî Liskov Substitution
- Elke animatie implementeert dezelfde interface: `init()`, `play()`, `pause()`, `reset()`, `destroy()`, `onComplete(cb)`
- Animaties zijn uitwisselbaar ‚Äî een eenvoudigere fallback kan altijd worden ingewisseld

### I ‚Äî Interface Segregation
- Engine biedt aparte API's voor navigatie, animatie-besturing, en progress-tracking
- Geen god-object ‚Äî elke module heeft een smalle API

### D ‚Äî Dependency Inversion
- Slides verwijzen naar animaties via `data-animation="naam"`, niet via directe imports
- Animatie-varianten via `data-animation-options` (bijv. A3: `{"mode": "leeg"}`, A10: `{"highlight": "verbinden"}`)
- Engine laadt animaties dynamisch op basis van configuratie
- Alle configuratie (slide-volgorde, timing, sprekers) in een centraal config-bestand

## Engine Specificatie

### Architectuur

De engine is een **SPA (Single Page Application)**. `index.html` is het enige HTML-bestand dat de browser laadt. Slides worden via `fetch()` ingeladen als HTML-fragmenten en in de DOM ge√Ønjecteerd.

```
index.html  ‚îÄ‚îÄloadSlide()‚îÄ‚îÄ‚ñ∫  fetch('slides/05-wat-is-meshcore.html')

                              slideContainer.innerHTML = response
                                    ‚îÇ

                                    ‚îÇ

                                    ‚îÇ
                                    ‚ñº
                              animation.init(container, options)
                                    ‚îÇ
                                    ‚ñº
                              animation.play()
```

### Slide-laden

1. Engine leest `config.js` bij opstarten
2. Bij navigatie: `fetch()` haalt slide-HTML op als fragment
3. Fragment wordt in `<main id="slide-container">` geplaatst
4. Vorige slide-animatie krijgt `destroy()` aangeroepen
5. Nieuwe animatie wordt dynamisch ge√Ømporteerd en ge√Ønitialiseerd
6. Fade-transitie (300ms) tijdens de wissel

### Keyboard & Clicker Shortcuts

| Toets | Actie |
|---|---|
| `‚Üí` / `Space` / `PageDown` / `N` | Volgende slide |
| `‚Üê` / `PageUp` / `P` | Vorige slide |
| `Home` | Eerste slide |
| `End` | Laatste slide |
| `F` | Fullscreen toggle |
| `Escape` | Fullscreen verlaten |
| `S` | Presenter-view openen (nieuw venster) |
| `T` | Timer starten/pauzeren |
| `B` | Scherm zwart (blackout) |

Wireless presenter/clicker stuurt standaard `PageDown`/`PageUp` ‚Äî deze zijn opgenomen in de shortcuts.

### Animatie-trigger Model

Elke slide heeft een `triggerMode` die bepaalt hoe navigatie en animatie samenwerken:

| `triggerMode` | Gedrag | Gebruiken bij |
|---|---|---|
| `'click'` | **Klik 1** = `animation.play()`. **Klik 2** = volgende slide. Als animatie al klaar is (`onComplete` gefired), gaat klik 1 direct door naar volgende slide. | Meeste slides met animatie |
| `'auto'` | Animatie start automatisch bij slide-laden. Klik = volgende slide (ongeacht animatie-status). | Wacht-slide (00), achtergrondanimaties |
| `'sequential'` | Meerdere animaties in volgorde. **Klik 1** = eerste animatie `play()`. **Klik 2** = tweede animatie `play()`. **Klik 3** = volgende slide. Enzovoort voor meer animaties. | Slide 16 (A9 ‚Üí A15) |
| `'none'` | Geen animatie. Klik = volgende slide. | Slides zonder animatie |

De engine leest `triggerMode` uit `config.js` per slide. Default: `'click'` als er een animatie is, `'none'` als er geen is.

### Custom Mesh-dashboard (slide 08)

De live demo op slide 08 gebruikt een **custom mesh-dashboard** dat in de presentatie-engine zelf is gebouwd ‚Äî geen externe app of website.

- **Implementatie:** speciale animatie `A17 mesh-dashboard` (`animations/mesh-dashboard.js`)
- **Functionaliteit:** toont live inkomende berichten, afzender-node, hop-count, en een visuele mesh-topologie
- **Data-bron:** connecteert met een lokale MeshCore companion-node via Web Bluetooth of Web Serial API
- **Fallback:** als connectie mislukt, toon een pre-recorded demo-sequentie (opgeslagen als `data/demo-messages.json`)
- **Layout:** B (fullscreen) ‚Äî dashboard vult het hele scherm

### Presenter-view Mechanisme

- **Openen:** druk `S` op het toetsenbord, of navigeer naar `index.html?mode=presenter`
- **Communicatie:** De presenter-view en het beamer-venster communiceren via `BroadcastChannel('meshcore-presentation')`
- **Sync:** navigatie in √©√©n venster wordt gesynchroniseerd naar het andere
- **Bestandsstructuur:** presenter-view layout zit in `engine.js` (geen apart HTML-bestand nodig ‚Äî het is dezelfde `index.html` met `?mode=presenter` die een ander layout rendert)
